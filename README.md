# codetranslate

`codetranslate` is a Python CLI tool that translates comments, docstrings, and Markdown documentation in codebases from one natural language to another. It uses Helsinki-NLP/OPUS-MT models via HuggingFace `transformers` and runs entirely locally (no API keys needed).
Translate comments, docstrings, and Markdown documentation in codebases from one natural language to another. Uses [Helsinki-NLP/OPUS-MT](https://huggingface.co/Helsinki-NLP) models via HuggingFace `transformers` — runs entirely locally with no API keys.


## Disclaimer

Currently in BETA. Translations are generated by open-source neural machine translation models (Helsinki-NLP/OPUS-MT). Therefore running this tool uses lots of memory and is not designed for large text documents. Some languages require an authorization token from HuggingFace. Quality varies by language pair and context — technical terms, idioms, and nuanced prose may not translate accurately. Always review translated output before using it in production.

## Development

```bash
pip install -e ".[dev]"
pytest tests/ -v
```

Requires Python 3.10+ and [PyTorch](https://pytorch.org/get-started/locally/). The first run will download the translation model (~300 MB), which is cached in `~/.cache/huggingface/` for subsequent use.

## Quickstart Usage

### Translate a local file or directory

```bash
# Translate a single file (overwrites in place)
codetranslate translate src/utils.py --to fr

# Translate a directory recursively
codetranslate translate src/ --to es

# Write translated files to a separate directory
codetranslate translate src/ --to fr --output src-fr/

# Preview without writing
codetranslate translate src/ --to es --dry-run
```

### Installation and Running Tests
```bash
# Install (editable, with dev deps)
pip install -e ".[dev]"

# Run tests
python -m pytest tests/ -v

# Run a single test file
python -m pytest tests/test_parsers.py -v

# Run a single test
python -m pytest tests/test_parsers.py::TestParsePythonFile::test_extracts_docstrings -v

# Run with coverage
python -m pytest tests/ --cov=code_translate

# Use the CLI — local path
codetranslate translate <path> --to <lang> [--from en] [--output <dir>] [--dry-run]

# Use the CLI — single file from GitHub (downloaded via raw API, no clone)
codetranslate translate https://github.com/user/repo/blob/main/README.md --from en --to fr
codetranslate translate https://github.com/user/repo/blob/main/README.md --from en --to fr --output ./README-fr.md
```

### Translate a GitHub repository

Pass any GitHub repo URL — it will be shallow-cloned automatically:

```bash
# Preview translations (dry-run by default for remote repos)
codetranslate translate https://github.com/pymumu/smartdns --from zh --to en

# Write translated files to a directory
codetranslate translate https://github.com/pymumu/smartdns --from zh --to en --output ../smartdns-en

# Keep the cloned repo for inspection
codetranslate translate https://github.com/pymumu/smartdns --from zh --to en --keep-temp
```

### List supported languages

```bash
codetranslate languages
```

### Options

| Flag | Description |
|------|-------------|
| `--to <code>` | Target language (required) |
| `--from <code>` | Source language (default: `en`) |
| `--output, -o <path>` | Output file or directory (default: overwrite in place; for remote files without `--output`, prints to stdout) |
| `--dry-run` | Preview translations without writing files |
| `--keep-temp` | Keep the temp clone when translating a remote repo |
| `--verbose, -v` | Show model name and detailed progress |
| `--no-cache` | Disable the disk cache; always re-translate from scratch |
| `--clear-cache` | Clear all cached translations from `~/.cache/codetranslate/` and exit |

### Translation cache

Translations are cached to disk at `~/.cache/codetranslate/`. Each translated chunk is stored as a file keyed by a SHA-256 hash of the model name and source text. This means:

- **Repeated runs are fast** — previously translated chunks are loaded from disk instead of re-running the model
- **Memory usage stays low** — only one chunk is in memory at a time during translation
- **Identical comments across files** (e.g. license headers, `# TODO:` notes) are translated once and reused
- **Interrupted runs can be resumed** — already-translated chunks are skipped

Use `--no-cache` to force fresh translations, or `--clear-cache` to delete the entire cache:

```bash
# Force fresh translations (ignore cache)
codetranslate translate src/ --to fr --no-cache

# Clear the cache and exit
codetranslate translate src/ --to fr --clear-cache
```

## Supported languages

Uses free Helsinki-NLP/OPUS-MT models which may change at any time. Not all languages are included automatically, they have to download upon execution.

| Code | Language |
|------|----------|
| `en` | English |
| `es` | Spanish |
| `fr` | French |
| `it` | Italian |
| `nl` | Dutch |
| `ru` | Russian |
| `sv` | Swedish |
| `uk` | Ukrainian |
| `vi` | Vietnamese |
| `zh` | Chinese |

The model is resolved as `Helsinki-NLP/opus-mt-{source}-{target}` — if a direct pair doesn't exist, check [Helsinki-NLP on HuggingFace](https://huggingface.co/Helsinki-NLP) for available models.
